BOM

מבוא ל-BOM

BOM — Browser Object Model — הוא מכלול האובייקטים שהדפדפן מספק ל-JavaScript לצורך אינטראקציה עם סביבת הדפדפן עצמו, מחוץ למסמך ה-HTML.

DOM

  • ייצוג של מסמך ה-HTML
  • אלמנטים, תוכן, סגנונות
  • הגדרה סטנדרטית (W3C)
  • document.querySelector(...)

BOM

  • ייצוג של חלון הדפדפן
  • URL, היסטוריה, מידע מכשיר
  • לא סטנדרטי לחלוטין (browser-specific)
  • window.location.href

האובייקט המרכזי של ה-BOM הוא window — האובייקט הגלובלי של הדפדפן. כל האובייקטים האחרים (location, history, navigator) הם מאפיינים של window.

השוואה ל-C#:
ב-C# WinForms: Application, Screen, Environment — מידע על סביבת ריצה.
ב-JS: window, navigator, screen — מידע על סביבת הדפדפן.

window

האובייקט window — האובייקט הגלובלי

window הוא האובייקט הגלובלי של JavaScript בדפדפן. כל משתנה גלובלי, כל פונקציה גלובלית, וכל האובייקטים המובנים — כולם מאפיינים של window.

// כל הדברים הבאים שקולים: console.log("hello"); window.console.log("hello"); setTimeout(() => {}, 1000); window.setTimeout(() => {}, 1000); // משתנה גלובלי הופך למאפיין של window: var myName = "יוסי"; console.log(window.myName); // "יוסי" // אובייקטים מובנים: window.document // ה-DOM window.location // כתובת URL window.history // היסטוריית ניווט window.navigator // מידע על הדפדפן window.screen // מידע על המסך window.localStorage // אחסון מקומי
ניתן לכתוב window.alert() או פשוט alert() — שתי הצורות זהות. הקידומת window. מושמטת בדרך כלל מפני שהיא מובנת מאליה.

window methods

פונקציות שימושיות של window

// alert — הצגת הודעה (חוסמת): alert("ברוכים הבאים!"); window.alert("הודעה חשובה"); // confirm — אישור כן/לא: const ok = confirm("האם להמשיך?"); if (ok) { console.log("המשתמש אישר"); } else { console.log("המשתמש ביטל"); } // prompt — קלט טקסטואלי: const name = prompt("מה שמך?", "אורח"); // ערך ברירת מחדל: "אורח" if (name) { console.log("שלום, " + name); } // open — פתיחת חלון/כרטיסייה חדשה: const newTab = window.open("https://example.com", "_blank"); // close — סגירת חלון (עובד רק על חלונות שנפתחו בקוד): newTab.close(); // print — פתיחת דיאלוג הדפסה: window.print();
alert, confirm ו-prompt חוסמים את ריצת הקוד עד לתגובת המשתמש. בפרויקטים מודרניים מומלץ להחליפם ב-UI מותאם אישית (modal, dialog).

מידות וגלילה

מידות ומאפייני חלון

window חושף מאפיינים המתארים את גודל אזור התצוגה ומיקום הגלילה — שימושיים לעיצוב responsive ולגלילה מבוקרת.

// גודל אזור התצוגה (viewport): console.log(window.innerWidth); // רוחב בפיקסלים console.log(window.innerHeight); // גובה בפיקסלים // מיקום גלילה נוכחי: console.log(window.scrollX); // גלילה אופקית console.log(window.scrollY); // גלילה אנכית (הנפוץ) // גלילה לראש הדף: window.scrollTo(0, 0); // גלילה חלקה: window.scrollTo({ top: 0, behavior: "smooth" }); // גלילה יחסית (מהמיקום הנוכחי): window.scrollBy(0, 300); // 300px למטה // דוגמה: כפתור "חזור לראש" שמופיע בגלילה: window.addEventListener("scroll", () => { const btn = document.getElementById("backToTop"); btn.style.display = window.scrollY > 400 ? "block" : "none"; }); // responsive — התאמה לגודל חלון: window.addEventListener("resize", () => { if (window.innerWidth < 768) { document.body.classList.add("mobile"); } else { document.body.classList.remove("mobile"); } });

location

האובייקט location

window.location מכיל את כל המידע על ה-URL הנוכחי של הדף, ומספק פונקציות לניווט פרוגרמטי.

// URL לדוגמה: // https://example.com:8080/shop/products?q=shoes&color=red#results console.log(location.href); // ה-URL המלא console.log(location.protocol); // "https:" console.log(location.hostname); // "example.com" console.log(location.port); // "8080" console.log(location.pathname); // "/shop/products" console.log(location.search); // "?q=shoes&color=red" console.log(location.hash); // "#results" console.log(location.host); // "example.com:8080"
location.search מחזיר את ה-query string כולל סימן ?. לניתוח הפרמטרים יש להשתמש ב-new URLSearchParams(location.search) כפי שהוסבר ביחידה הקודמת.
השוואה ל-C#:
ב-ASP.NET: Request.Url — גישה לחלקי ה-URL בצד השרת.
ב-JS: window.location — גישה לאותו מידע בצד הלקוח, בזמן אמת.

location methods

פונקציות ומאפיינים של location

// ניווט לדף אחר (נשמר ב-history): location.href = "https://example.com/new-page"; // שקול ל: location.assign("https://example.com/new-page"); // ניווט ללא שמירה ב-history (לא ניתן "חזרה"): location.replace("https://example.com/login"); // שימוש נפוץ: redirect לאחר logout // רענון הדף: location.reload(); // רגיל location.reload(true); // כפיית טעינה מהשרת (עוקף cache) // שינוי חלק ספציפי ב-URL: location.pathname = "/about"; // ניווט ל-/about location.search = "?page=2"; // עדכון query string location.hash = "#section3"; // גלילה לעוגן (ללא טעינה מחדש) // בדיקת דף נוכחי: if (location.pathname === "/dashboard") { loadDashboardData(); }
ההבדל בין assign ל-replace הוא קריטי: assign שומר את הדף הנוכחי ב-history (ניתן לחזור), replace מחליף את הרשומה הנוכחית (לא ניתן לחזור).

history

האובייקט history

window.history מאפשר גישה לרשימת הדפים שנצפו בכרטיסייה הנוכחית ומספק פונקציות לניווט ביניהם — בלי לחשוף את כתובות הדפים עצמן (מסיבות פרטיות).

// מספר הרשומות ב-history: console.log(history.length); // מספר דפים בהיסטוריה // חזרה לדף הקודם (כמו כפתור Back בדפדפן): history.back(); // שקול ל: history.go(-1); // קדימה לדף הבא: history.forward(); history.go(1); // קפיצה למספר דפים: history.go(-3); // 3 דפים אחורה history.go(2); // 2 דפים קדימה // pushState — הוספת רשומה ל-history ללא טעינת דף: history.pushState({ page: "products" }, "", "/products"); // URL משתנה ל-/products, הדף לא נטען מחדש // replaceState — החלפת הרשומה הנוכחית: history.replaceState({ page: "cart" }, "", "/cart"); // האזנה לשינוי ב-history (לחיצת Back/Forward): window.addEventListener("popstate", (e) => { console.log("מצב:", e.state); renderPage(e.state); });
pushState ו-replaceState הם הבסיס לכל ה-SPA Routers (React Router, Vue Router) — הם מאפשרים ניווט ב-URL מבלי לטעון מחדש את הדף.

navigator

האובייקט navigator

window.navigator מספק מידע על הדפדפן, מערכת ההפעלה, השפה, ומצב הרשת של המשתמש.

// שפת הדפדפן: console.log(navigator.language); // "he-IL", "en-US", "ar"... console.log(navigator.languages); // ["he-IL", "he", "en-US", "en"] // מחרוזת זיהוי הדפדפן (User Agent): console.log(navigator.userAgent); // "Mozilla/5.0 (Windows NT 10.0; Win64; x64) Chrome/..." // מצב חיבור לרשת: console.log(navigator.onLine); // true / false // פלטפורמה (מיושן בחלקו): console.log(navigator.platform); // "Win32", "MacIntel"... // מספר ליבות מעבד: console.log(navigator.hardwareConcurrency); // 8 // האם מגע נתמך: console.log(navigator.maxTouchPoints); // 0 = אין, >0 = touch // Geolocation — מיקום המשתמש (דורש הרשאה): navigator.geolocation.getCurrentPosition( (pos) => { console.log("קו רוחב:", pos.coords.latitude); console.log("קו אורך:", pos.coords.longitude); }, (err) => console.error("שגיאת מיקום:", err.message) );

דוגמאות מעשיות

דוגמאות מעשיות — שילוב אובייקטי דפדפן

זיהוי שפת הדפדפן והתאמת תוכן

function applyLanguage() { const lang = navigator.language.split("-")[0]; // "he", "en", "ar" if (lang === "he" || lang === "ar") { document.dir = "rtl"; document.documentElement.lang = lang; } else { document.dir = "ltr"; document.documentElement.lang = lang; } }

בדיקת מצב רשת עם עדכון חי

function updateOnlineStatus() { const banner = document.getElementById("offlineBanner"); if (!navigator.onLine) { banner.style.display = "block"; banner.innerText = "אין חיבור לאינטרנט"; } else { banner.style.display = "none"; } } window.addEventListener("online", updateOnlineStatus); window.addEventListener("offline", updateOnlineStatus); updateOnlineStatus(); // בדיקה ראשונית

שמירת מיקום גלילה בין טעינות

window.addEventListener("scroll", () => { localStorage.setItem("scrollPos", window.scrollY); }); document.addEventListener("DOMContentLoaded", () => { const pos = Number(localStorage.getItem("scrollPos")) || 0; window.scrollTo({ top: pos, behavior: "instant" }); });

תרגול מסכם

תרגול מסכם — שילוב אובייקטי הדפדפן

תרחיש משולב: אפליקציה שמשתמשת ב-window, location, history ו-navigator יחד.

// --- אתחול האפליקציה --- document.addEventListener("DOMContentLoaded", () => { // 1. התאמת שפה לפי הדפדפן: const userLang = navigator.language.startsWith("he") ? "he" : "en"; document.documentElement.lang = userLang; document.dir = userLang === "he" ? "rtl" : "ltr"; // 2. קריאת פרמטר מה-URL (עמוד נוכחי בפגינציה): const params = new URLSearchParams(location.search); const page = Number(params.get("page")) || 1; loadPage(page); // 3. שמירת מיקום גלילה: const savedScroll = Number(localStorage.getItem("scroll")) || 0; window.scrollTo(0, savedScroll); // 4. כפתורי ניווט — שימוש ב-history: document.getElementById("prevPage").addEventListener("click", () => { if (page > 1) { history.pushState({}, "", "?page=" + (page - 1)); location.reload(); } }); document.getElementById("nextPage").addEventListener("click", () => { history.pushState({}, "", "?page=" + (page + 1)); location.reload(); }); // 5. שמירת גלילה לפני עזיבת הדף: window.addEventListener("beforeunload", () => { localStorage.setItem("scroll", window.scrollY); }); // 6. בדיקת חיבור רשת: if (!navigator.onLine) { alert("שים לב: אין חיבור לאינטרנט. חלק מהתוכן עשוי לא להיטען."); } });
סיכום — מתי משתמשים בכל אחד:
window — גודל חלון, גלילה, דיאלוגים, פתיחת כרטיסייה.
location — ניווט בין דפים, קריאת URL, redirect.
history — ניווט אחורה/קדימה, SPA routing.
navigator — שפה, מצב רשת, מידע על מכשיר.
