חזרה קצרה על DOM

להלן סיכום המושגים המרכזיים מהיחידות הקודמות, כבסיס להמשך:

// בחירת אלמנט בודד: const el = document.querySelector(".card"); // בחירת מספר אלמנטים: const items = document.querySelectorAll("li"); // יצירת אלמנט חדש: const div = document.createElement("div"); div.innerText = "תוכן"; div.classList.add("box"); // הוספה ל-DOM: document.getElementById("container").append(div); // קריאת ערך מ-input: const val = document.getElementById("myInput").value; // האזנה לאירוע click: el.addEventListener("click", () => { el.classList.toggle("active"); });

אירועים

מבוא לאירועים ב-JavaScript

אירוע (Event) הוא אות שמופק על ידי הדפדפן בתגובה לפעולה — של המשתמש, של הרשת, או של המסמך עצמו. JavaScript מאפשר להגדיר קוד שירוץ בדיוק כאשר אירוע מסוים מתרחש.

תכנות מבוסס אירועים (Event-driven programming) הוא הבסיס לכל ממשק משתמש ב-web. הקוד אינו רץ ברצף קבוע — הוא מחכה לאירועים ומגיב אליהם.

אירועי משתמש

  • לחיצה על כפתור
  • הקלדה בשדה
  • גלילה בעמוד
  • מעבר עכבר

אירועי מערכת

  • סיום טעינת הדף
  • סיום בקשת רשת
  • שינוי גודל חלון
  • שגיאות בזמן ריצה
השוואה ל-C#:
ב-C# WinForms: button1.Click += Handler; — חיבור handler לאירוע ספציפי.
ב-JS: btn.addEventListener("click", handler); — אותו מודל מחשבתי, תחביר שונה.

Event Listeners

מאזינים לאירועים — addEventListener

addEventListener היא הדרך המומלצת לחיבור קוד לאירוע. היא מאפשרת הפרדה מלאה בין HTML לבין JavaScript, ותומכת בחיבור מספר handlers לאותו אירוע.

// תחביר מלא: element.addEventListener(eventType, callbackFunction, useCapture); // דוגמאות נפוצות: const btn = document.getElementById("myBtn"); // פונקציה אנונימית: btn.addEventListener("click", function() { console.log("נלחץ!"); }); // Arrow function: btn.addEventListener("click", () => { btn.style.backgroundColor = "teal"; }); // פונקציה מוגדרת נפרד: function handleClick() { console.log("handler חיצוני"); } btn.addEventListener("click", handleClick); // חיבור מספר listeners לאותו אלמנט — אפשרי: btn.addEventListener("click", logTime); btn.addEventListener("click", updateUI);
יש להימנע משימוש ב-onclick="..." ישירות ב-HTML — גישה זו מערבת תוכן ולוגיקה, ומגבילה לhandler אחד בלבד.

אירועי עכבר

אירועי עכבר

הדפדפן מפיק אירועים שונים בהתאם לפעולת העכבר מעל אלמנט:

const box = document.getElementById("box"); // לחיצה בודדת: box.addEventListener("click", () => console.log("click")); // לחיצה כפולה: box.addEventListener("dblclick", () => console.log("dblclick")); // כניסת עכבר לאלמנט (לא עולה לאבות — לא מבעבע): box.addEventListener("mouseenter", () => box.classList.add("hover")); // יציאת עכבר מהאלמנט: box.addEventListener("mouseleave", () => box.classList.remove("hover")); // תנועה של עכבר מעל האלמנט (מופעל ברציפות!): box.addEventListener("mousemove", (e) => { console.log("X:", e.clientX, "Y:", e.clientY); });
mouseenter לעומת mouseover:
mouseenter — מופעל פעם אחת בכניסה לאלמנט, אינו מבעבע.
mouseover — מופעל גם בכניסה לילדים, מבעבע לאלמנטי-אב. לרוב עדיף mouseenter.

אירועי מקלדת

אירועי מקלדת

אירועי מקלדת מאפשרים תגובה להקלדת המשתמש בזמן אמת:

const input = document.getElementById("searchInput"); // keydown — מופעל ברגע לחיצה על מקש (לפני הכנסת התו): input.addEventListener("keydown", (e) => { console.log("מקש שנלחץ:", e.key); // זיהוי מקש Enter: if (e.key === "Enter") { performSearch(); } // זיהוי Escape: if (e.key === "Escape") { input.value = ""; } }); // keyup — מופעל לאחר שחרור המקש (התו כבר הוכנס): input.addEventListener("keyup", (e) => { console.log("ערך נוכחי:", input.value); });
keypress הוא אירוע ישן שסומן כמיושן (deprecated) ואין להשתמש בו בקוד חדש. יש להשתמש ב-keydown או keyup במקומו.

e.key מחזיר את ייצוג המקש כמחרוזת: "Enter", "Escape", "a", "ArrowUp" וכדומה.

טעינה וגלילה

אירועי טעינה וגלילה

DOMContentLoaded

מופעל ברגע שה-HTML פורס במלואו, לפני שתמונות וסגנונות חיצוניים נטענו. זהו המקום הנכון להפעיל קוד שמצפה שה-DOM יהיה מוכן.

document.addEventListener("DOMContentLoaded", () => { // ה-DOM מוכן — ניתן לבחור אלמנטים בבטחה const btn = document.getElementById("startBtn"); btn.addEventListener("click", init); });

load

מופעל רק לאחר טעינה מלאה של כל משאבי הדף (תמונות, גופנים, scripts).

window.addEventListener("load", () => { console.log("הדף נטען במלואו"); });

scroll

window.addEventListener("scroll", () => { const pos = window.scrollY; if (pos > 300) { document.getElementById("backToTop").style.display = "block"; } });
כלל מעשי:
יש להשתמש ב-DOMContentLoaded לרוב המקרים — הוא מהיר יותר מ-load ומספיק לעבודה עם DOM.

טפסים

מבוא לאירועי טפסים

טפסים הם הדרך העיקרית לקבלת קלט מהמשתמש. ב-web מודרני, JavaScript שולט באופן מלא בטיפול בנתוני הטופס — בלי טעינת דף מחדש.

אירועי טפסים מאפשרים להגיב לכל שינוי, למנוע שליחה שגויה, ולוודא שהנתונים תקינים לפני שהם מועברים הלאה.

אירועים נפוצים

  • focus — שדה נבחר
  • blur — שדה עזב
  • input — כל שינוי בזמן אמת
  • change — שינוי לאחר יציאה
  • submit — שליחת טופס

שימושים נפוצים

  • ולידציה בזמן אמת
  • חיפוש חי (live search)
  • הגדרת ערכי ברירת מחדל
  • מניעת שליחה כפולה
  • הצגת חיווי שגיאה
השוואה ל-C#:
ב-WinForms: textBox1.TextChanged += Handler;
ב-JS: input.addEventListener("input", handler); — אותו מודל, ביצוע בדפדפן.

focus ו-blur

אירועי focus ו-blur

focus מופעל כאשר אלמנט מקבל פוקוס — לחיצה עליו או Tab. blur מופעל כאשר האלמנט מאבד פוקוס — עזיבת השדה.

const emailInput = document.getElementById("email"); const errorMsg = document.getElementById("emailError"); // כניסה לשדה — ניקוי שגיאות קודמות: emailInput.addEventListener("focus", () => { emailInput.classList.remove("error"); errorMsg.innerText = ""; }); // יציאה מהשדה — בדיקת תקינות: emailInput.addEventListener("blur", () => { const val = emailInput.value.trim(); if (!val.includes("@")) { emailInput.classList.add("error"); errorMsg.innerText = "כתובת דוא\"ל אינה תקינה"; } });

focus

  • ניקוי הודעות שגיאה
  • הדגשת שדה פעיל
  • טעינת נתונים דינמיים

blur

  • ולידציה לאחר הקלדה
  • שמירת ערך אוטומטית
  • הסתרת רשימת השלמה

input ו-change

אירועי input ו-change — השוואה מפורטת

שני האירועים קשורים לשינויים בשדות קלט, אך עם הבדל מהותי בתזמון:

input

מופעל בכל שינוי — כל הקלדת תו, מחיקה, הדבקה.

field.addEventListener("input", () => { counter.innerText = field.value.length + " תווים"; });

change

מופעל רק לאחר יציאה מהשדה אם הערך השתנה.

select.addEventListener("change", () => { filterProducts(select.value); });
// input — מתאים ל: חיפוש חי, ספירת תווים, תצוגה מיידית const search = document.getElementById("search"); search.addEventListener("input", () => { renderResults(search.value); }); // change — מתאים ל: dropdown, checkbox, radio, select const category = document.getElementById("category"); category.addEventListener("change", () => { loadCategory(category.value); });
לשדות input type="text" — יש להשתמש ב-input לתגובה מיידית.
לשדות select, checkbox, radio — change הוא הבחירה הטבעית.

submit מול click

submit מול לחיצה על כפתור

הבדל זה הוא אחד הנושאים הנפוצים לבלבול. submit הוא אירוע של הטופס, לא של הכפתור. הוא מופעל גם ב-Enter בשדה, לא רק בלחיצה על כפתור.

// האזנה לאירוע submit — עדיפה: const form = document.getElementById("loginForm"); form.addEventListener("submit", (e) => { e.preventDefault(); // מניעת טעינת דף מחדש — קריטי! const username = document.getElementById("username").value; const password = document.getElementById("password").value; if (!username || !password) { showError("יש למלא את כל השדות"); return; } sendToServer({ username, password }); });
// האזנה ל-click בלבד — חסרה: // לא תתפוס Enter בשדה, ולא תתפוס submit מוגדר אחרת. btn.addEventListener("click", () => { // פחות מומלץ לטיפול בטפסים });
preventDefault — למה הוא קריטי:
ברירת המחדל של submit היא ניווט לעמוד חדש ושליחת נתוני הטופס כ-GET/POST. e.preventDefault() מבטל פעולה זו ומאפשר לטפל בנתונים ב-JavaScript.

event object

אובייקט האירוע (Event Object)

כל handler מקבל אוטומטית פרמטר — ה-event object. אובייקט זה מכיל מידע מלא על האירוע שהתרחש.

btn.addEventListener("click", (e) => { console.log(e.type); // "click" — סוג האירוע console.log(e.target); // האלמנט שנלחץ בפועל console.log(e.currentTarget); // האלמנט שה-listener מחובר אליו console.log(e.clientX); // מיקום X של העכבר console.log(e.clientY); // מיקום Y של העכבר console.log(e.timeStamp); // זמן האירוע }); // event object באירוע מקלדת: input.addEventListener("keydown", (e) => { console.log(e.key); // "Enter", "a", "Escape"... console.log(e.shiftKey); // true/false — Shift לחוץ? console.log(e.ctrlKey); // true/false — Ctrl לחוץ? e.preventDefault(); // מניעת פעולת ברירת מחדל }); // target לעומת currentTarget: const ul = document.querySelector("ul"); ul.addEventListener("click", (e) => { console.log(e.target); // ה-li שנלחץ console.log(e.currentTarget); // ה-ul שה-listener מחובר אליו });
e.target ו-e.currentTarget יהיו שונים כאשר האירוע מבעבע (bubbling). לרוב, בעבודה עם event delegation, מתעניינים ב-e.target.
שקופית 14 מתוך 17
Bubbling

Event Bubbling — בעבוע אירועים

כאשר אירוע מתרחש על אלמנט, הוא מופץ כלפי מעלה בעץ ה-DOM לאלמנטי-האב, עד לאובייקט document. תופעה זו נקראת bubbling.

// מבנה HTML: // <div id="outer"> // <div id="inner"> // <button id="btn">לחץ</button> // </div> // </div> document.getElementById("btn").addEventListener("click", () => { console.log("1. כפתור"); }); document.getElementById("inner").addEventListener("click", () => { console.log("2. inner div"); }); document.getElementById("outer").addEventListener("click", () => { console.log("3. outer div"); }); // לחיצה על הכפתור תדפיס: // 1. כפתור → 2. inner div → 3. outer div

עצירת bubbling

btn.addEventListener("click", (e) => { e.stopPropagation(); // האירוע לא יגיע לאלמנטי-האב console.log("רק הכפתור"); });
Event Delegation — שימוש מעשי ב-bubbling:
ניתן לחבר listener אחד לאלמנט-אב, ולזהות e.target כדי לטפל בכל הילדים — יעיל במיוחד עבור רשימות ורכיבים שנוצרים דינמית.

תרגול

תרגול DOM — מערך מוצרים

יצירת כרטיסי מוצר דינמיים ממערך נתונים, תוך שימוש ב-createElement, append ואירועים:

const products = [ { id: 1, name: "מחשב נייד", price: 3200, category: "electronics" }, { id: 2, name: "עכבר אלחוטי", price: 89, category: "electronics" }, { id: 3, name: "כיסא משרדי", price: 650, category: "furniture" }, ]; function renderProducts(list) { const container = document.getElementById("products"); container.innerHTML = ""; // ניקוי לפני רינדור list.forEach(product => { const card = document.createElement("div"); card.classList.add("product-card"); card.dataset.id = product.id; const name = document.createElement("h3"); name.innerText = product.name; const price = document.createElement("p"); price.innerText = product.price + " ₪"; const btn = document.createElement("button"); btn.innerText = "הוסף לסל"; btn.addEventListener("click", () => addToCart(product)); card.append(name, price, btn); container.append(card); }); } renderProducts(products);

סינון

סינון מוצרים — חיפוש וקטגוריה

שילוב סינון לפי טקסט חופשי וסינון לפי קטגוריה, עם רינדור מחדש דינמי:

const products = [ /* ... */ ]; const searchInput = document.getElementById("search"); const categorySelect = document.getElementById("category"); function getFiltered() { const term = searchInput.value.toLowerCase().trim(); const cat = categorySelect.value; // "all" או קטגוריה ספציפית return products.filter(p => { const matchText = p.name.toLowerCase().includes(term); const matchCat = cat === "all" || p.category === cat; return matchText && matchCat; }); } function update() { const filtered = getFiltered(); renderProducts(filtered); const msg = document.getElementById("noResults"); msg.style.display = filtered.length === 0 ? "block" : "none"; } // שני מקורות סינון — אותה פונקציה: searchInput.addEventListener("input", update); categorySelect.addEventListener("change", update);
חשוב לשים לב: כל שינוי — בחיפוש או בקטגוריה — מפעיל את אותה פונקציה update שמחשבת מחדש את הסינון ומרנדרת מחדש את התוצאות. זהו דפוס נפוץ ויעיל לניהול מצב (state) בממשק משתמש.

תרגול מסכם

תרגול מסכם מתקדם — מנהל משימות

תרחיש אמיתי המשלב DOM, אירועים, טפסים, מערכים ורינדור דינמי:

let tasks = []; const form = document.getElementById("taskForm"); const input = document.getElementById("taskInput"); const list = document.getElementById("taskList"); const counter = document.getElementById("taskCount"); // הוספת משימה — דרך submit: form.addEventListener("submit", (e) => { e.preventDefault(); const text = input.value.trim(); if (!text) return; tasks.push({ id: Date.now(), text, done: false }); input.value = ""; render(); }); function render() { list.innerHTML = ""; tasks.forEach(task => { const li = document.createElement("li"); li.innerText = task.text; if (task.done) li.classList.add("done"); // סימון כבוצע בלחיצה: li.addEventListener("click", () => { task.done = !task.done; render(); }); // כפתור מחיקה: const del = document.createElement("button"); del.innerText = "מחק"; del.addEventListener("click", (e) => { e.stopPropagation(); // מניעת bubbling ל-li tasks = tasks.filter(t => t.id !== task.id); render(); }); li.appendChild(del); list.appendChild(li); }); counter.innerText = "סה\"כ: " + tasks.length + " משימות"; }
עיקרון מנחה:
המבנה: state (מערך) → render (DOM) → events (עדכון state) → render הוא הבסיס לכל framework מודרני (React, Vue). הבנתו ב-vanilla JS היא תשתית חיונית.
