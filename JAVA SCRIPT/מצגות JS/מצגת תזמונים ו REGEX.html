Timers

מבוא לטיימרים ב-JavaScript

טיימרים מאפשרים לדחות ביצוע קוד לזמן עתידי, או להפעיל קוד בצורה מחזורית. הם מהווים את הכלי הבסיסי לעבודה אסינכרונית ב-JavaScript — קוד שמתבצע לאחר זמן מה, מבלי לחסום את המשך ריצת התוכנית.

שימושים נפוצים

  • הצגת הודעה לאחר השהייה
  • ספירה לאחור
  • עדכון שעון/תאריך
  • אנימציות מבוססות זמן
  • שמירה אוטומטית (autosave)

שתי הפונקציות המרכזיות

  • setTimeout — ביצוע חד-פעמי לאחר השהייה
  • setInterval — ביצוע חוזר במרווחים קבועים
השוואה ל-C#:
ב-C#: Task.Delay(1000) או System.Timers.Timer
ב-JS: setTimeout ו-setInterval — קלים לשימוש, מובנים בשפה, ללא צורך ב-async/await לשימוש בסיסי.

setTimeout

setTimeout — ביצוע מושהה

setTimeout(callback, delay) מפעיל פונקציה פעם אחת, לאחר השהייה של delay מילישניות.

// פונקציה אנונימית — השהייה של שנייה: setTimeout(function() { console.log("עבר שנייה!"); }, 1000); // Arrow function — נקי יותר: setTimeout(() => { document.getElementById("msg").innerText = "ההודעה הופיעה!"; }, 2000); // פונקציה מוגדרת נפרד: function showWelcome() { alert("ברוכים הבאים!"); } setTimeout(showWelcome, 3000); // העברת פרמטרים לפונקציה: function greet(name, title) { console.log("שלום " + title + " " + name); } setTimeout(greet, 1000, "כהן", "ד\"ר"); // delay של 0 — ביצוע אחרי הקוד הסינכרוני הנוכחי: setTimeout(() => console.log("אחרון"), 0); console.log("ראשון"); // יודפס לפני!
הערך המוחזר מ-setTimeout הוא מזהה מספרי (timer ID). שמירתו מאפשרת ביטול הטיימר לפני שהופעל, באמצעות clearTimeout.

setInterval

setInterval — ביצוע מחזורי

setInterval(callback, interval) מפעיל פונקציה שוב ושוב, כל interval מילישניות — עד שמבטלים אותו ידנית.

// הדפסה כל שנייה: setInterval(() => { console.log(new Date().toLocaleTimeString("he-IL")); }, 1000); // עדכון שעון בדף: setInterval(() => { const now = new Date(); document.getElementById("clock").innerText = now.toLocaleTimeString("he-IL"); }, 1000); // ספירה עולה: let count = 0; setInterval(() => { count++; document.getElementById("counter").innerText = count; }, 500);

setTimeout

  • מופעל פעם אחת
  • לאחר השהייה שהוגדרה
  • מסתיים מעצמו

setInterval

  • מופעל שוב ושוב
  • כל X מילישניות
  • דורש ביטול ידני
השוואה ל-C#:
ב-C#: new System.Timers.Timer(1000) { AutoReset = true }
ב-JS: setInterval(() => { ... }, 1000) — פשוט יותר, ללא הגדרת אובייקט.

עצירת טיימרים

עצירת טיימרים — clearTimeout ו-clearInterval

כל טיימר מחזיר מזהה מספרי (ID). שמירת ה-ID מאפשרת ביטול הטיימר בכל עת. טיימרים שלא בוטלו ממשיכים לפעול — גם כשאינם נחוצים, וגורמים לדליפת זיכרון.

// ביטול setTimeout: const timeoutId = setTimeout(() => { alert("לא תראה הודעה זו"); }, 5000); // ביטול לפני שהופעל: clearTimeout(timeoutId); // ביטול setInterval: let seconds = 10; const countdownId = setInterval(() => { document.getElementById("timer").innerText = seconds; seconds--; if (seconds < 0) { clearInterval(countdownId); // עצירה כשמסתיים document.getElementById("timer").innerText = "הסתיים!"; } }, 1000); // ביטול על ידי לחיצת כפתור: let intervalId = null; document.getElementById("startBtn").addEventListener("click", () => { intervalId = setInterval(updateClock, 1000); }); document.getElementById("stopBtn").addEventListener("click", () => { clearInterval(intervalId); });
חשוב לשמור תמיד את ה-ID של טיימרים שנוצרים — במיוחד ב-setInterval. טיימר שלא נעצר ממשיך לפעול ברקע גם לאחר שהאלמנטים שלו הוסרו מה-DOM.

RegExp

מבוא ל-Regular Expressions

ביטוי רגולרי (Regular Expression / RegExp) הוא תבנית לתיאור מחרוזות. הוא מאפשר לחפש, לאמת ולהחליף טקסט לפי כללים מדויקים — ללא צורך בלולאות וקוד מסועף.

שימושים נפוצים

  • ולידציית אימייל, טלפון, סיסמה
  • חיפוש תבנית בטקסט
  • החלפת חלקי מחרוזת
  • פיצול מחרוזות לפי תבנית
  • ניתוח קלט משתמש

שני תחבירים

// תחביר literal (נפוץ): const re = /hello/; // תחביר constructor: const re2 = new RegExp("hello");
השוואה ל-C#:
ב-C#: new Regex(@"\d+") מ-System.Text.RegularExpressions
ב-JS: RegExp מובנה בשפה, תחביר ה-literal /pattern/flags נפוץ ונוח יותר.

תחביר

תחביר בסיסי של RegExp

תווים מיוחדים

  • \d — ספרה (0–9)
  • \w — אות, ספרה, קו תחתון
  • \s — רווח / טאב
  • . — כל תו (חוץ מ-newline)
  • ^ — תחילת מחרוזת
  • $ — סוף מחרוזת

כמויות (Quantifiers)

  • * — 0 פעמים ומעלה
  • + — פעם אחת ומעלה
  • ? — 0 או 1 פעמים
  • {3} — בדיוק 3 פעמים
  • {2,5} — בין 2 ל-5
  • {3,} — לפחות 3
// קבוצות תווים (Character Sets): /[abc]/ // a או b או c /[a-z]/ // כל אות קטנה /[A-Z]/ // כל אות גדולה /[0-9]/ // כל ספרה (שקול ל-\d) /[^0-9]/ // כל תו שאינו ספרה (^ בתוך [] = שלילה) // Flags — דגלים: /pattern/i // case-insensitive /pattern/g // global — מצא את כולם /pattern/m // multiline // דוגמאות: /^\d{3}-\d{7}$/ // טלפון: 3 ספרות, מקף, 7 ספרות /^[a-zA-Z0-9]+$/ // אותיות וספרות בלבד

פונקציות מחרוזת

פונקציות על מחרוזות עם RegExp

const text = "JavaScript הוא שפת תכנות מספר 1 בעולם"; // match — מחזיר מערך של תוצאות: console.log(text.match(/\d+/g)); // ["1"] console.log(text.match(/Java\w*/g)); // ["JavaScript"] // search — מחזיר אינדקס של התוצאה הראשונה (או -1): console.log(text.search(/\d+/)); // 36 console.log(text.search(/Python/)); // -1 // replace — החלפת תוצאות: const clean = text.replace(/\d+/g, "X"); // "JavaScript הוא שפת תכנות מספר X בעולם" // החלפת שם עם פונקציה: const upper = text.replace(/\b\w+/g, w => w.toUpperCase()); // split — פיצול לפי תבנית: const csv = "אחד, שתיים, שלוש,ארבע"; const parts = csv.split(/\s*,\s*/); // פיצול עם רווחים אופציונליים // ["אחד", "שתיים", "שלוש", "ארבע"] // includes — בדיקה פשוטה (ללא RegExp): console.log(text.includes("JavaScript")); // true

RegExp.test

RegExp.test — בדיקת תאימות

regex.test(string) בודק האם המחרוזת תואמת לתבנית, ומחזיר true או false. זוהי הדרך הנפוצה והיעילה ביותר לולידציה.

// בדיקות בסיסיות: const onlyDigits = /^\d+$/; console.log(onlyDigits.test("12345")); // true console.log(onlyDigits.test("123ab")); // false console.log(onlyDigits.test("")); // false // בדיקת אורך מינימלי: const minLength = /^.{8,}$/; console.log(minLength.test("hello")); // false (7 תווים) console.log(minLength.test("hello123")); // true // שימוש ישיר ב-literal: console.log(/^[a-z]+$/.test("hello")); // true console.log(/^[a-z]+$/.test("Hello")); // false // שימוש בפונקציית ולידציה: function isValidInput(value, pattern) { return pattern.test(value.trim()); } const result = isValidInput(" 123 ", /^\d+$/); console.log(result); // true (trim מוסר רווחים)
test הוא המתודה העיקרית לולידציה — פשוטה, מהירה, ומחזירה boolean ישירות. לעומת match שמחזיר את הנתונים שנמצאו, test רק שואל "האם קיים?"

ולידציות

ולידציות נפוצות באמצעות RegExp

// אימייל — בסיסי: const emailRe = /^[^\s@]+@[^\s@]+\.[^\s@]{2,}$/; // ^[^\s@]+ — תווים לפני ה-@ (לא רווח, לא @) // @ — סימן החובה // [^\s@]+ — שם הדומיין // \. — נקודה (חייבת להיות escaped) // [^\s@]{2,}$ — סיומת דומיין (לפחות 2 תווים) emailRe.test("user@example.com"); // true emailRe.test("userexample.com"); // false (אין @) emailRe.test("user@.com"); // false // טלפון ישראלי: const phoneRe = /^0\d{1,2}-?\d{7}$/; phoneRe.test("052-1234567"); // true phoneRe.test("0521234567"); // true phoneRe.test("123456"); // false // סיסמה חזקה (אות גדולה + קטנה + ספרה + מינימום 8): const passRe = /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,}$/; // (?=...) — Lookahead: מוודא שהתנאי קיים איפשהו // מספרים בלבד: const numRe = /^\d+$/; // שם (עברית ואנגלית, רווחים): const nameRe = /^[\u0590-\u05FFa-zA-Z\s]{2,}$/;
(?=...) הוא Lookahead — בודק שתנאי מסוים קיים בהמשך המחרוזת, מבלי לצרוך תווים. משמש ליצירת כללי סיסמה מורכבים.

pattern ב-HTML

מאפיין pattern בטפסי HTML

מאפיין pattern מאפשר הגדרת RegExp ישירות על שדה input ב-HTML. הדפדפן מבצע ולידציה אוטומטית בעת שליחת הטופס — ללא JavaScript.

<!-- ולידציית מספר טלפון --> <input type="text" id="phone" pattern="0\d{1,2}-?\d{7}" title="יש להזין מספר טלפון תקין (לדוגמה: 052-1234567)" required /> <!-- ולידציית מספרים בלבד --> <input type="text" pattern="[0-9]+" title="ספרות בלבד" /> <!-- ולידציית סיסמה --> <input type="password" pattern="(?=.*[A-Z])(?=.*\d).{8,}" title="לפחות 8 תווים, אות גדולה וספרה" />
pattern לעומת JavaScript validation:
pattern — קל ומהיר, ולידציה אוטומטית של הדפדפן, ללא JS.
JS validation — גמיש יותר, מאפשר הודעות שגיאה מותאמות אישית ולוגיקה מורכבת. בפרויקטים אמיתיים — משתמשים בשתיהן יחד.

תרגול מסכם

תרגול מסכם — טופס עם ולידציה וטיימר

שילוב: שדות קלט, ולידציה ב-RegExp, ומשוב מושהה בטיימר.

const RULES = { email: /^[^\s@]+@[^\s@]+\.[^\s@]{2,}$/, phone: /^0\d{1,2}-?\d{7}$/, pass: /^(?=.*[A-Z])(?=.*\d).{8,}$/ }; function validate(fieldId, pattern) { const input = document.getElementById(fieldId); const err = document.getElementById(fieldId + "Error"); const val = input.value.trim(); const valid = pattern.test(val); input.classList.toggle("valid", valid); input.classList.toggle("invalid", !valid); err.style.display = valid ? "none" : "block"; return valid; } // ולידציה בזמן אמת: document.getElementById("email") .addEventListener("blur", () => validate("email", RULES.email)); document.getElementById("phone") .addEventListener("blur", () => validate("phone", RULES.phone)); document.getElementById("pass") .addEventListener("blur", () => validate("pass", RULES.pass)); // שליחת טופס עם משוב מושהה: document.getElementById("form").addEventListener("submit", (e) => { e.preventDefault(); const allValid = validate("email", RULES.email) && validate("phone", RULES.phone) && validate("pass", RULES.pass); if (allValid) { const msg = document.getElementById("success"); msg.style.display = "block"; setTimeout(() => { msg.style.display = "none"; }, 4000); } });
