querySelectorAll

מבוא ל-querySelectorAll

בעוד ש-querySelector מחזיר אלמנט בודד (הראשון שנמצא), querySelectorAll מחזיר את כל האלמנטים התואמים לבורר CSS בצורת NodeList — אוסף סטטי של צמתים.

// מחזיר NodeList של כל האלמנטים עם class="card" const cards = document.querySelectorAll(".card"); console.log(cards); // NodeList(3) [div, div, div] console.log(cards.length); // 3 console.log(cards[0]); // האלמנט הראשון
  • אם לא נמצאו אלמנטים — מוחזרת NodeList ריקה (לא null)
  • הבורר יכול להיות כל בורר CSS תקין: tag, class, id, מורכב
  • הערכים מסודרים לפי סדר הופעתם במסמך
השוואה ל-C#:
ב-C# (LINQ): controls.OfType<Button>() מחזיר אוסף של רכיבים מסוג מסוים.
ב-JS: querySelectorAll("button") — אותו רעיון, על בסיס בוררי CSS.

querySelectorAll

עבודה עם querySelectorAll

ניתן להשתמש בבוררים מורכבים כדי לשלוף אלמנטים לפי מבנה מקונן:

// כל כפתורים בתוך div עם id="menu" const btns = document.querySelectorAll("#menu button"); // כל li עם class="active" const activeItems = document.querySelectorAll("li.active"); // שני בוררים יחד (פסיק — union) const all = document.querySelectorAll("h2, h3"); // גישה לאיבר לפי אינדקס console.log(btns[0]); // הכפתור הראשון console.log(btns[btns.length - 1]); // האחרון
ה-NodeList שמוחזרת היא סטטית — אינה מתעדכנת אוטומטית אם ה-DOM משתנה לאחר השליפה. בניגוד לכך, getElementsByClassName מחזיר אוסף חי (live collection) שמשתנה עם ה-DOM.
השוואה ל-C#:
גישה לפי אינדקס: list[0] — זהה בתחביר.
ב-JS: nodeList[0] — אותה גישה, על NodeList.

לולאות

ריצה על אוסף אלמנטים

יש מספר דרכים לעבור על כל אלמנטי ה-NodeList:

לולאת for קלאסית

const items = document.querySelectorAll(".item"); for (let i = 0; i < items.length; i++) { items[i].style.color = "teal"; }

לולאת for...of

for (const item of items) { item.classList.add("highlighted"); }

forEach

items.forEach((item, index) => { item.innerText = "פריט מספר " + (index + 1); });
השוואה ל-C#:
ב-C#: foreach (var item in items) { ... }
ב-JS: for...of — תחביר כמעט זהה.
forEach מקביל ל-.ToList().ForEach() ב-LINQ.

NodeList vs Array

השוואה בין NodeList למערך רגיל

NodeList נראית כמו מערך אך אינה מערך מלא. היא תומכת ב-forEach ובגישה לפי אינדקס, אך חסרות בה פונקציות רבות של Array.

זמין ב-NodeList

items.length items[0] items.forEach() items.item(0)

זמין ב-Array בלבד

items.map() // X items.filter() // X items.find() // X items.reduce() // X

המרה למערך מלא

// דרך 1 — Array.from const arr = Array.from(document.querySelectorAll(".item")); // דרך 2 — spread operator const arr2 = [...document.querySelectorAll(".item")]; // כעת זמינות כל פונקציות המערך: const texts = arr.map(el => el.innerText); const visible = arr.filter(el => el.style.display !== "none");
השוואה ל-C#:
IEnumerable מול List<T> — דומה מאוד.
Array.from(nodeList) מקביל ל-.ToList() ב-LINQ.
שקופית 7 מתוך 14
this

this בהקשר ל-DOM

כאשר משתמשים בפונקציה רגילה (לא arrow function) כ-event handler, this בתוך הפונקציה מצביע על האלמנט שהפעיל את האירוע.

const buttons = document.querySelectorAll("button"); buttons.forEach(function(btn) { btn.addEventListener("click", function() { // this = הכפתור הספציפי שנלחץ this.style.backgroundColor = "teal"; this.innerText = "נלחץ!"; }); });

שימוש זה מאפשר לכתוב handler אחד שפועל על מספר אלמנטים, כאשר כל אחד מטפל בעצמו בלבד — ללא צורך לזהות איזה אלמנט נלחץ.

חשוב לשים לב: ערך this ב-JavaScript נקבע בזמן ריצה לפי אופן הקריאה לפונקציה, לא לפי מיקום הגדרתה. בפונקציות חץ, this מתנהג אחרת — ראו שקופית הבאה.
השוואה ל-C#:
ב-C#: this תמיד מצביע על המחלקה הנוכחית — ערך קבוע ויציב.
ב-JS: this הוא דינמי — ערכו נקבע לפי ההקשר בזמן הקריאה.
שקופית 8 מתוך 14
Arrow Function and this

פונקציות חץ והקשר ל-this

פונקציית חץ אינה יוצרת הקשר this משלה — היא יורשת את this מהסביבה החיצונית שבה הוגדרה. לכן כ-event handler, this לא יצביע על האלמנט.

// פונקציה רגילה — this = האלמנט שנלחץ btn.addEventListener("click", function() { console.log(this); // <button>...</button> }); // Arrow function — this אינו האלמנט! btn.addEventListener("click", () => { console.log(this); // Window (או undefined ב-strict mode) });

הפתרון: שימוש ב-event object

btn.addEventListener("click", (e) => { // e.currentTarget = האלמנט שה-listener מחובר אליו e.currentTarget.style.backgroundColor = "teal"; // e.target = האלמנט שנלחץ בפועל // (עשוי להיות שונה בעת bubble של אירוע) console.log(e.target); });
כלל מעשי:
בשימוש ב-arrow function — יש להשתמש ב-e.currentTarget במקום this.
בשימוש בפונקציה רגילה — ניתן להשתמש ב-this ישירות.

DOM Traversal

טיול על ה-DOM (DOM Traversal)

לאחר בחירת אלמנט, ניתן לנוע ממנו לאלמנטים קרובים בעץ דרך מאפיינים מובנים — ללא חיפוש מחדש ב-document.

const item = document.querySelector(".selected"); // אלמנט אב: item.parentElement; // ילדים ישירים: item.children; // HTMLCollection — רק אלמנטים item.childNodes; // NodeList — כולל טקסט ו-whitespace // אחים: item.nextElementSibling; item.previousElementSibling; // ילד ראשון / אחרון: item.firstElementChild; item.lastElementChild; //מביא את אלמנט ה אב הקרוב לפי הסלקטור הנשלח item.closest(selector) — מחפש את האלמנט הקרוב ביותר שמתאים לבחירה.

דוגמת שימוש — סימון ילדי רשימה

const list = document.getElementById("myList"); Array.from(list.children).forEach((child, i) => { child.classList.add("visible"); child.dataset.index = i; // שמירת נתון ב-data attribute });
השוואה ל-C#:
ב-WPF: VisualTreeHelper.GetParent(element)
ב-JS: element.parentElement — ישיר ופשוט יותר.

יצירת אלמנטים

createElement — יצירת אלמנטים חדשים

document.createElement יוצר אלמנט HTML חדש בזיכרון. האלמנט אינו מוצג בדף עד שמוסיפים אותו ל-DOM בפירוש.

// יצירת אלמנטים: const newDiv = document.createElement("div"); const newBtn = document.createElement("button"); const newP = document.createElement("p"); // הגדרת תוכן: newP.innerText = "פסקה שנוצרה דינמית"; // הגדרת מאפיינים: newBtn.innerText = "לחץ כאן"; newBtn.id = "myNewBtn"; newBtn.classList.add("primary-btn"); // הגדרת style: newDiv.style.backgroundColor = "#e8f4f4"; newDiv.style.padding = "12px"; newDiv.style.borderRadius = "8px"; // ניתן לחבר אירוע לפני הוספה לדף: newBtn.addEventListener("click", () => { alert("כפתור נוצר דינמית!"); });
השוואה ל-C#:
WinForms: Button btn = new Button(); btn.Text = "לחץ"; this.Controls.Add(btn);
JS: const btn = document.createElement("button"); btn.innerText = "לחץ"; container.append(btn);
הדפוס זהה — יצירה, הגדרת מאפיינים, הוספה למיכל.

הוספה ל-DOM

append ו-appendChild — הוספה ל-DOM

לאחר יצירת אלמנט, יש להוסיפו ל-DOM כדי שיופיע בדף:

const container = document.getElementById("container"); const newItem = document.createElement("li"); newItem.innerText = "פריט חדש"; // appendChild — ותיק יותר, מקבל Node אחד, מחזיר אותו container.appendChild(newItem); // append — מודרני, מקבל מספר ארגומנטים וגם מחרוזות container.append(newItem); container.append("טקסט ישיר", anotherNode); // prepend — הוספה בתחילת המיכל container.prepend(newItem); // insertBefore — הוספה לפני אלמנט קיים const ref = document.getElementById("refItem"); container.insertBefore(newItem, ref);

append (מודרני)

// Nodes + מחרוזות // מספר ארגומנטים // ללא ערך חוזר

appendChild (ותיק)

// Node בלבד // ארגומנט אחד // מחזיר את ה-Node
השוואה ל-C#:
WinForms: this.Controls.Add(btn);
JS: container.append(btn); — דפוס זהה.

מבנה דינמי

בניית מבנה דינמי

ניתן לבנות רשימות ומבנים שלמים בלולאה, תוך שילוב של יצירת אלמנטים, הגדרת מאפיינים וחיבור אירועים — לפני ההוספה ל-DOM.

const data = ["ינואר", "פברואר", "מרץ", "אפריל"]; const ul = document.createElement("ul"); data.forEach((month, index) => { const li = document.createElement("li"); li.innerText = month; li.dataset.index = index; // data attribute li.addEventListener("click", function() { this.classList.toggle("selected"); }); ul.appendChild(li); // מוסיף ל-ul בזיכרון }); // הוספה ל-DOM פעם אחת — יעיל! document.getElementById("container").append(ul);
מומלץ לבנות את כל המבנה בזיכרון ולהוסיפו בפעולה אחת ל-DOM. הוספה של כל אלמנט בנפרד גורמת ל-reflow חוזר ופוגעת בביצועים.
השוואה ל-C#:
WinForms: listBox1.Items.AddRange(array);
JS: בניית אלמנטים בלולאה + append פעם אחת — אותו עיקרון של batch update.

תרגול משולב

תרגול משולב — שליפה, אירועים, יצירה

תרגיל זה משלב שליפת אלמנטים, האזנה לאירועים, קבלת קלט ויצירת אלמנטים חדשים ב-DOM.

// HTML: // <input id="taskInput" placeholder="הכנס משימה" /> // <button id="addBtn">הוסף</button> // <ul id="taskList"></ul> const addBtn = document.getElementById("addBtn"); const input = document.getElementById("taskInput"); const list = document.getElementById("taskList"); addBtn.addEventListener("click", () => { const text = input.value.trim(); if (!text) return; const li = document.createElement("li"); li.innerText = text; const del = document.createElement("button"); del.innerText = "X"; del.addEventListener("click", () => li.remove()); li.appendChild(del); list.appendChild(li); input.value = ""; input.focus(); });

תרגול מתקדם

תרגול מתקדם — רשימה דינמית עם סינון

תרחיש אמיתי: רשימת פריטים עם יכולת סינון חי לפי קלט המשתמש. שילוב של querySelectorAll, forEach, ערכי input ועדכון ה-DOM.

// HTML: // <input id="search" placeholder="חפש..." /> // <ul id="itemList"> // <li>תפוח</li><li>בננה</li><li>תפוז</li> // </ul> // <p id="noResults" style="display:none">אין תוצאות</p> const search = document.getElementById("search"); search.addEventListener("input", () => { const term = search.value.toLowerCase().trim(); const items = document.querySelectorAll("#itemList li"); items.forEach(item => { const match = item.innerText.toLowerCase().includes(term); item.style.display = match ? "list-item" : "none"; }); const visible = [...items].filter(i => i.style.display !== "none"); const msg = document.getElementById("noResults"); if (msg) msg.style.display = visible.length === 0 ? "block" : "none"; });
עיקרון מנחה לסיכום:
כל אפליקציית web בנויה על שלושה שלבים: בחר אלמנט — האזן לאירוע — שנה את ה-DOM.
querySelectorAll, forEach, createElement ו-append הם הכלים הבסיסיים ליישום עיקרון זה.
