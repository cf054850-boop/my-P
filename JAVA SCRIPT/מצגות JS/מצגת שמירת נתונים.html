שמירת נתונים

מבוא לשמירת נתונים בדפדפן

כברירת מחדל, כל נתון שנוצר ב-JavaScript קיים רק בזיכרון — וניכחד עם רענון הדף. לעיתים קרובות יש צורך לשמור מידע כך שיישאר זמין גם לאחר מכן.

מידע זמני

  • מצב ממשק משתמש
  • פריטים בסל קניות
  • מצב חיפוש פעיל
  • נתוני סשן נוכחי

מידע קבוע

  • העדפות משתמש
  • ערכת עיצוב (dark/light)
  • שפה נבחרת
  • פרטי כניסה לא-רגישים

הדפדפן מציע מנגנוני שמירה מובנים שאינם דורשים שרת: localStorage, sessionStorage, ו-Cookies. בשיעור זה נתמקד בשניים הראשונים.

השוואה ל-C#:
ב-C# שמירת נתונים בין הפעלות דורשת קובץ, מסד נתונים, או Registry.
בדפדפן — localStorage מאפשר שמירה מקומית פשוטה, ללא שרת וללא קבצים.

Storage

localStorage ו-sessionStorage

שני מנגנוני ה-Storage פועלים באותו API בדיוק, אך נבדלים בחיי המידע:

מאפיין localStorage sessionStorage
משך החיים ללא הגבלה (עד מחיקה ידנית) עד סגירת הכרטיסייה
שיתוף בין כרטיסיות כן — כל כרטיסיות ה-origin לא — כרטיסייה נוכחית בלבד
נגישות אחרי רענון כן כן
נגישות אחרי סגירת דפדפן כן לא
שימוש אופייני העדפות, מצב קבוע נתוני סשן, טפסים זמניים
שני הסוגים שומרים נתונים כמחרוזות בלבד. שמירת אובייקטים ומערכים דורשת המרה ל-JSON — נושא שיוסבר בהמשך.
שקופית 5 מתוך 17
setItem

שמירת נתונים — setItem

setItem(key, value) שומר זוג מפתח-ערך ב-Storage. אם המפתח כבר קיים — הערך הישן מוחלף.

// שמירה ב-localStorage (קבועה): localStorage.setItem("username", "yossi"); localStorage.setItem("theme", "dark"); localStorage.setItem("fontSize", "16"); // שמירה ב-sessionStorage (זמנית): sessionStorage.setItem("currentStep", "2"); sessionStorage.setItem("formDraft", "טקסט שהוזן עד כה");
חשוב לשים לב: כל ערך נשמר כמחרוזת. גם המספר 16 יישמר כ-"16". בעת שליפה יש לטפל בהמרת הטיפוס אם נדרש.
השוואה ל-C#:
ב-C#: Properties.Settings.Default["theme"] = "dark";
ב-JS: localStorage.setItem("theme", "dark");
הרעיון זהה — מפתח ייחודי מזהה את הערך.
שקופית 6 מתוך 17
getItem

שליפת נתונים — getItem

getItem(key) מחזיר את הערך השמור תחת המפתח הנתון. אם המפתח אינו קיים — מוחזר null.

// שליפה פשוטה: const username = localStorage.getItem("username"); console.log(username); // "yossi" // טיפול במפתח לא קיים: const lang = localStorage.getItem("language"); console.log(lang); // null // שימוש עם ערך ברירת מחדל: const theme = localStorage.getItem("theme") || "light"; console.log(theme); // "light" אם לא נשמר // המרת מחרוזת למספר: const size = localStorage.getItem("fontSize"); const sizeNum = Number(size); // המרה ידנית נדרשת console.log(typeof sizeNum); // "number"
בעבודה עם ערכים מספריים או בוליאניים, יש לבצע המרת טיפוס ידנית לאחר שליפה: Number(), parseInt(), או value === "true" לבוליאניים.

סוגריים []

גישה בסוגריים — תחביר חלופי

בנוסף ל-setItem ו-getItem, ניתן לגשת ל-Storage באמצעות תחביר סוגריים — בדומה לגישה למאפיין של אובייקט.

// תחביר setItem: localStorage.setItem("color", "blue"); // תחביר סוגריים — שקול לחלוטין: localStorage["color"] = "blue"; // שליפה: console.log(localStorage.getItem("color")); // "blue" console.log(localStorage["color"]); // "blue" // גם dot notation אפשרי (אך פחות מומלץ): localStorage.color = "blue"; console.log(localStorage.color); // "blue"
ההמלצה המקצועית:
יש להשתמש ב-setItem ו-getItem כדרך הראשית. תחביר הסוגריים נוח לכתיבה מהירה, אך עלול להסתיר שגיאות — לדוגמה, כתיבה ל-localStorage.length עלולה לגרום להתנהגות בלתי צפויה.
שקופית 8 מתוך 17
מחיקה

מחיקה וניקוי נתונים

JavaScript מספק שתי פעולות מחיקה ב-Storage: מחיקה של מפתח ספציפי, וניקוי כל הנתונים של ה-origin.

// מחיקת מפתח ספציפי: localStorage.removeItem("username"); // בדיקה שהמפתח אכן נמחק: console.log(localStorage.getItem("username")); // null // מחיקת כל הנתונים של האתר: localStorage.clear(); // בדיקת מספר פריטים שמורים: console.log(localStorage.length); // 0 // דוגמה מעשית — התנתקות משתמש: function logout() { localStorage.removeItem("authToken"); localStorage.removeItem("userId"); sessionStorage.clear(); // ניקוי גם מה-session window.location.href = "/login.html"; }
clear() מוחק את כל המפתחות השמורים של האתר הנוכחי. יש להשתמש בו בזהירות — במיוחד כאשר מספר חלקים באפליקציה משתמשים ב-Storage.

JSON

מבוא ל-JSON

JSON (JavaScript Object Notation) הוא פורמט טקסטואלי לייצוג נתונים מובנים. הוא קריא לאדם, קל לפענוח על ידי מכונה, ומשמש כשפה משותפת להעברת נתונים בין שרתים, APIs, ורכיבי אפליקציה.

אובייקט JavaScript

const user = { name: "יוסי", age: 28, active: true, tags: ["admin","user"] };

מחרוזת JSON

'{ "name": "יוסי", "age": 28, "active": true, "tags": ["admin","user"] }'
  • כל המפתחות חייבים להיות מוקפים במרכאות כפולות
  • ערכים תקינים: מחרוזת, מספר, בוליאני, null, מערך, אובייקט
  • לא נתמכים: פונקציות, undefined, Date (ישיר)
השוואה ל-C#:
JSON ב-C# טופל דרך System.Text.Json או Newtonsoft.Json.
ב-JS — JSON מובנה בשפה עצמה דרך JSON.stringify ו-JSON.parse.
שקופית 10 מתוך 17
JSON.stringify

JSON.stringify — אובייקט למחרוזת

JSON.stringify(value) ממיר אובייקט JavaScript למחרוזת JSON. הפעולה נדרשת לפני כל שמירה ב-Storage או שליחה לשרת.

// אובייקט פשוט: const product = { id: 1, name: "מחשב", price: 3200 }; const str = JSON.stringify(product); console.log(str); // '{"id":1,"name":"מחשב","price":3200}' console.log(typeof str); // "string" // מערך של אובייקטים: const cart = [ { id: 1, qty: 2 }, { id: 3, qty: 1 } ]; const cartStr = JSON.stringify(cart); console.log(cartStr); // '[{"id":1,"qty":2},{"id":3,"qty":1}]' // עם עיצוב קריא (לצורכי debug): const pretty = JSON.stringify(product, null, 2); console.log(pretty); // { // "id": 1, // "name": "מחשב", // "price": 3200 // }
ערכי undefined ופונקציות מושמטים אוטומטית מהתוצאה. יש לוודא שכל הנתונים שברצון לשמור הם ערכי JSON תקינים.
שקופית 11 מתוך 17
JSON.parse

JSON.parse — מחרוזת לאובייקט

JSON.parse(string) ממיר מחרוזת JSON חזרה לאובייקט JavaScript. פעולה זו נדרשת לאחר כל שליפה מ-Storage או קבלת תגובה מ-API.

// שליפה מ-Storage והמרה: const raw = localStorage.getItem("product"); // raw = '{"id":1,"name":"מחשב","price":3200}' const product = JSON.parse(raw); console.log(product.name); // "מחשב" console.log(product.price); // 3200 (מספר, לא מחרוזת) console.log(typeof product); // "object" // מערך — parse מחזיר מערך: const cartRaw = localStorage.getItem("cart"); const cart = JSON.parse(cartRaw); console.log(Array.isArray(cart)); // true console.log(cart[0].qty); // 2 // טיפול במקרה שהנתון לא קיים: const saved = localStorage.getItem("settings"); const settings = saved ? JSON.parse(saved) : {};
הזרימה המלאה:
שמירה: אובייקט → JSON.stringify() → localStorage.setItem()
שליפה: localStorage.getItem() → JSON.parse() → אובייקט

שמירה מורכבת

שמירת מידע מורכב ב-localStorage

שמירת אובייקטים ומערכים דורשת תמיד את הזרימה: stringify → setItem → getItem → parse. להלן דוגמה מלאה ומעשית:

// --- שמירת רשימת משימות --- let tasks = [ { id: 1, text: "ללמוד JavaScript", done: false }, { id: 2, text: "לסיים את הפרויקט", done: true } ]; // שמירה: function saveTasks() { localStorage.setItem("tasks", JSON.stringify(tasks)); } // שליפה עם ערך ברירת מחדל: function loadTasks() { const raw = localStorage.getItem("tasks"); return raw ? JSON.parse(raw) : []; } // שימוש: tasks = loadTasks(); // בטעינת הדף tasks.push({ id: 3, text: "משימה חדשה", done: false }); saveTasks(); // אחרי כל שינוי // --- שמירת הגדרות משתמש --- const prefs = { theme: "dark", lang: "he", fontSize: 16 }; localStorage.setItem("prefs", JSON.stringify(prefs)); const savedPrefs = JSON.parse(localStorage.getItem("prefs") || "{}"); document.body.className = savedPrefs.theme || "light";

URL Parameters

מבוא להעברת נתונים ב-URL

Query Parameters הם נתונים שמועברים כחלק מה-URL עצמו, לאחר סימן השאלה ?. זוהי דרך נפוצה ומובנית להעביר מידע בין דפים, או לשמר מצב הניתן לשיתוף ולשמירה.

// מבנה URL עם parameters: // https://example.com/results?q=javascript&lang=he&page=2 // פירוט: // ? — תחילת ה-query string // q=javascript — פרמטר ראשון: מפתח=ערך // & — מפריד בין פרמטרים // lang=he — פרמטר שני // page=2 — פרמטר שלישי

שימושים נפוצים

  • שדות חיפוש (?q=...)
  • סינון וסידור תוצאות
  • מעבר בין עמודים (pagination)
  • העברת id לעמוד פרטים

יתרונות

  • ניתן לשתף ולשמור כ-bookmark
  • גלוי ושקוף
  • נתמך ב-back/forward
  • לא דורש Storage
השוואה ל-C#:
ב-ASP.NET: Request.QueryString["q"]
ב-JS (צד לקוח): new URLSearchParams(location.search).get("q")

יצירת URL

יצירת URL עם נתונים

בעת בניית URL עם נתונים שהוזנו על ידי משתמש, יש לקודד ערכים מיוחדים כדי למנוע שבירה של מבנה ה-URL. לכך משמש encodeURIComponent.

// בניית URL ידנית (פחות מומלצת): const name = "יוסי כהן"; const city = "תל אביב"; const url = "profile.html?name=" + name + "&city=" + city; // בעיה: רווחים ותווים מיוחדים שוברים את ה-URL // בניית URL עם קידוד נכון: const safeUrl = "profile.html" + "?name=" + encodeURIComponent(name) + "&city=" + encodeURIComponent(city); // profile.html?name=%D7%99%D7%95%D7%A1%D7%99%20%D7%9B%D7%94%D7%9F&city=... // שיטה מומלצת — URLSearchParams: const params = new URLSearchParams(); params.set("name", name); params.set("city", city); params.set("id", 42); const cleanUrl = "profile.html?" + params.toString(); console.log(cleanUrl); // profile.html?name=%D7%99%D7%95%D7%A1%D7%99+%D7%9B%D7%94%D7%9F&city=... // ניווט לדף עם הנתונים: window.location.href = cleanUrl;
URLSearchParams מטפל בקידוד אוטומטית ומספק API נוח לבניית query strings. יש להעדיפו על פני שרשור ידני של מחרוזות.

שליפה מ-URL

שליפת נתונים מה-URL — URLSearchParams

URLSearchParams מאפשר קריאה נוחה של פרמטרים מה-URL הנוכחי או מכל מחרוזת query string.

// URL נוכחי: profile.html?name=יוסי&city=תל+אביב&id=42 // שליפת כל הפרמטרים: const params = new URLSearchParams(window.location.search); // קריאת ערך בודד: const name = params.get("name"); console.log(name); // "יוסי" // קריאת מספר: const id = Number(params.get("id")); console.log(id); // 42 // בדיקת קיום פרמטר: if (params.has("city")) { const city = params.get("city"); console.log("עיר:", city); // "תל אביב" } // ערך ברירת מחדל למקרה שהפרמטר חסר: const page = Number(params.get("page")) || 1; // מעבר על כל הפרמטרים: params.forEach((value, key) => { console.log(key + " = " + value); });
params.get(key) מחזיר null אם הפרמטר אינו קיים. יש תמיד לטפל במקרה זה כדי למנוע שגיאות בזמן ריצה.

דוגמה מעשית

דוגמה מעשית מלאה — טופס ודף תוצאה

תרחיש: טופס משתמש בדף הרשמה שמעביר נתונים לדף אישור דרך URL.

דף הטופס (register.html)

const form = document.getElementById("registerForm"); form.addEventListener("submit", (e) => { e.preventDefault(); const name = document.getElementById("nameInput").value.trim(); const email = document.getElementById("emailInput").value.trim(); const plan = document.getElementById("planSelect").value; if (!name || !email) { alert("יש למלא את כל השדות"); return; } // בניית URL עם נתונים: const params = new URLSearchParams({ name, email, plan }); window.location.href = "confirm.html?" + params.toString(); });

document.addEventListener("DOMContentLoaded", () => { const params = new URLSearchParams(window.location.search); const name = params.get("name") || "אורח"; const email = params.get("email") || "לא צוין"; const plan = params.get("plan") || "חינמי"; document.getElementById("welcomeMsg").innerText = "שלום, " + name + "! ההרשמה לתוכנית "" + plan + "" הושלמה."; document.getElementById("emailDisplay").innerText = email; // שמירת פרטים גם ב-localStorage: localStorage.setItem("user", JSON.stringify({ name, email, plan })); });

תרגול מסכם

תרגול מסכם — אפליקציית רשימת קניות

תרחיש משולב: רשימת קניות עם שמירה ב-localStorage, JSON וסינון לפי URL.

// --- ניהול רשימת קניות עם שמירה קבועה --- const STORAGE_KEY = "shoppingList"; function loadList() { const raw = localStorage.getItem(STORAGE_KEY); return raw ? JSON.parse(raw) : []; } function saveList(list) { localStorage.setItem(STORAGE_KEY, JSON.stringify(list)); } function addItem(name, category) { const list = loadList(); list.push({ id: Date.now(), name, category, done: false }); saveList(list); render(); } function toggleItem(id) { const list = loadList().map(item => item.id === id ? { ...item, done: !item.done } : item ); saveList(list); render(); } function render() { // סינון לפי URL parameter: const params = new URLSearchParams(window.location.search); const filterCat = params.get("category") || "all"; let list = loadList(); if (filterCat !== "all") { list = list.filter(i => i.category === filterCat); } const container = document.getElementById("list"); container.innerHTML = ""; list.forEach(item => { const li = document.createElement("li"); li.innerText = item.name + " [" + item.category + "]"; if (item.done) li.classList.add("done"); li.addEventListener("click", () => toggleItem(item.id)); container.append(li); }); } document.addEventListener("DOMContentLoaded", render);
עיקרון מנחה לסיכום:
השילוב localStorage + JSON + URL Parameters מאפשר אפליקציות עם מצב שמור, ניתן לשיתוף, ועובד לחלוטין ללא שרת — בסיס חיוני לכל פרויקט web מעשי.
